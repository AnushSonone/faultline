## Executive verdict

**`M2 PASSED WITH REQUIRED CLEANUP`**

The M2 product foundation is real: synthetic multi-signal Parquet fixture → canonical envelopes → session replay clock → cursor-filtered precomputed projections over WebSocket → topology / timeline / heatmap / trace UI with linked selection. It was visually convincing before this pass, but several acceptance gaps made M2 architecturally incomplete (unstable topology re-layout, seek-back traces ignoring cursor, WS gap without resync, multi-play tick races, fmt/clippy red, thin E2E). Required cleanup for those gaps is included in this audit branch. M3 must not start until this branch is accepted.

Do not describe `rec-mem-001` as a public real-world RCAEval incident. It is a committed synthetic Online Boutique-style fixture generated by project code.

## 2. Vertical slice evidence

| Step | Implementation | Data character | Tests | Gaps / fragility |
|------|----------------|----------------|-------|------------------|
| Fixture / dataset | `datasets/fixtures/synthetic-ob/v1/rec-mem-001`, `python/faultline_data/generate_fixture.py` | **Synthetic** project fixture (not RCAEval, not OTel recording) | Python validate; regenerate script | Real RE2-OB adapter still stub (`adapters/rcaeval.py`) |
| Canonical envelopes | `crates/common`, `crates/replay/src/reader.rs` | Parquet `payload_json` → `TelemetryEnvelope` | common + reader fixture load | `observed_time_ns == event_time_ns` at read; ingest overwrites `ingest_time_ns` with sequence placeholder |
| Normalized storage | Parquet partitions + `manifest.json` / `labels.json` | Fixture-derived, checksummed | catalog + python validate | Large real datasets not yet ingested |
| Replay controls | `crates/replay/src/clock.rs`, API play/pause/seek/speed/reset | Virtual event-time clock | clock unit tests + API smoke | Rapid play previously spawned multiple tick loops (fixed via `playback_epoch`) |
| Backend ingestion | `crates/ingest` | Bounded channel, dedupe | ingest unit tests | **Filled at load**; projections do **not** consume the channel in M2 |
| Session state | `crates/api/src/sessions.rs` | In-memory `HashMap` of sessions | API integration seek-back test | No session TTL / eviction |
| WebSocket projections | `crates/api/src/websocket.rs`, `crates/projection/*` | **Precomputed** full snapshots each tick (typed as snapshot/append/delta) | protocol gap → resync path; seek-back rebuild | Message type names `timeline.append` / `heatmap.delta` carry full payloads |
| Topology / timeline / heatmap / traces | `web/src/views/*` | Fixture-derived projections | Playwright shell + vitest store | Heatmap averages mem/lat/err into one cell value |
| Linked selection | `web/src/state/investigation.ts` | Shared Zustand store | vitest + E2E selection bar | Service filter on traces is weak; selection cleared on load |

### M2 → M3 boundary (precompute)

M2 rebuilds views by scanning `Session.envelopes` filtered by `clock.current_event_time_ns()` inside `publish_projections()`. There is **no** Arrow operator DAG, watermark service, or streaming query engine. M3 (TA-021+) must replace this scan/precompute path with ingest reorder/watermarks → Arrow batches → operators → projection emitters, without changing the WS DTO contract more than necessary.
